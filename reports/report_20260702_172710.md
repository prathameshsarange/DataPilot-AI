*   **Week 1-2: Python & Django/Flask Refresh**

*   **Week 3-4: SQL & Database Interaction**
    *   **Focus:** Learn SQL fundamentals (SELECT, INSERT, UPDATE, DELETE, JOINs, aggregations) with PostgreSQL. Understand relational database design. Practice integrating SQL directly into a Python application, then extensively use Django's ORM for database interactions.
    *   **Resources:** SQL tutorials (e.g., freeCodeCamp, W3Schools SQL), "PostgreSQL Tutorial" website, Django ORM documentation.
    *   **Project Goal:** Refactor the "EduTrack" project to use a proper PostgreSQL database via Django ORM. Add features like filtering by multiple criteria and basic user profiles.

*   **Month 2 (Days 31-60): Advanced Frontend & API Consumption**
    *   **Focus:** Deep dive into modern JavaScript (ES6+ features like Promises, Async/Await, modules). Master React.js fundamentals: components, props, state, hooks (useState, useEffect, useContext), and React Router. Learn to consume REST APIs from a React frontend using `fetch` or `Axios`.

    *   **Project Goal:** Build a full-stack "Recipe Sharing" application. Create a Django REST Framework backend for managing recipes (CRUD) and users. Develop a React frontend to display, search, and allow users to add/edit recipes, connecting via REST APIs.

*   **Month 3 (Days 61-90): Authentication, Testing & Deployment Basics**

Introduce unit testing for both Python backend (Pytest) and React frontend (Jest/React Testing Library). Learn Docker basics to containerize your Django application.     *   **Resources:** Django REST Framework documentation (Authentication & Permissions), Pytest/Jest documentation, Docker docs for beginners, Heroku/Vercel deployment guides.
    *   **Project Goal:** Enhance the "Recipe Sharing" application with secure user registration/login. Add a "Favorites" feature for logged-in users. Containerize the backend with Docker and deploy both frontend and backend to a free/starter cloud service. Document the deployment process.

# 🎤 Interview Preparation

**10 Technical Questions:**

1.  Explain the Model-View-Controller (MVC) architecture pattern and how Django adapts it (Model-View-Template).
2.  What is the difference between Django and Flask? When would you choose one over the other for a project?
3.  Describe the request-response cycle in a typical Django application.
4.  What is an ORM (Object-Relational Mapper), and how do you use Django's ORM to interact with a database? Provide an example.
5.  Explain the concept of RESTful APIs. How would you design a simple REST API for managing users?
6.  How do you handle user authentication and authorization in a Django application? What are tokens versus sessions?
7.  What is a virtual environment in Python, and why is it crucial for development?
8.  Explain the core principles of Git. Describe the workflow you follow for a typical feature development (e.g., branching, committing, merging).
9.  What are the key differences between `props` and `state` in React? When would you use each?
10. How would you debug a Python Django application that is encountering an `IntegrityError` when trying to save data?

**5 HR Questions:**

1.  As a current B.Tech student, how do you manage your academic workload alongside your personal projects and self-learning?
2.  What sparked your interest in Python and Full Stack Development, and how do you stay updated with new technologies in this field?
3.  Describe a time when you faced a significant challenge or obstacle in one of your projects. How did you approach it, and what was the outcome?
4.  Why are you seeking an internship at this stage of your education, and what specific contributions do you hope to make to our team?
5.  Where do you envision yourself professionally in the next three to five years, and how do you believe this internship will help you achieve those goals?

**5 Behavioral Questions:**

1.  Tell me about a time you had to learn a new programming language or framework very quickly for a project. What was your learning process?
2.  Describe a situation where you worked as part of a team on a technical project. What was your role, and how did you contribute to the team's success?
3.  Give an example of a time you received constructive criticism on your code or project. How did you react, and what did you learn?
4.  Recall a project where you encountered a major bug or unexpected issue. How did you go about identifying and resolving it?
5.  How do you prioritize tasks when working on multiple projects with competing deadlines? Provide an example.

**Common Mistakes:**
*   **Lack of Specificity:** Providing vague answers instead of concrete examples (especially for behavioral questions).
*   **Poor Code Explanation:** Struggling to articulate the thought process or design choices behind project code.
*   **Not Asking Questions:** Failing to ask thoughtful questions at the end of the interview, indicating a lack of engagement.
*   **Understating or Overstating Skills:** Not accurately representing their current skill level. It's okay to say "I'm learning X" or "I have basic knowledge of Y."
*   **Failing to Research:** Not knowing anything about the company, its products, or its tech stack.

**Interview Tips:**
*   **Research Thoroughly:** Understand the company's mission, products, and recent news. Look for their tech blog or GitHub if available.
*   **Practice Explaining Projects:** Be able to clearly articulate the purpose, challenges, technologies, and *your specific contributions* to each project on your resume.
*   **STAR Method:** Use the Situation, Task, Action, Result method to structure your answers for behavioral questions.
*   **Demonstrate Learnability:** As a student, emphasize your eagerness to learn, your quick learning ability, and your proactive approach to acquiring new skills.
*   **Prepare Questions:** Have 2-3 intelligent questions ready to ask the interviewer about the role, the team, the company culture, or their tech stack.
*   **Technical Problem-Solving:** For technical questions, walk the interviewer through your thought process even if you don't know the full answer. Show how you'd approach the problem.
*   **Follow Up:** Send a polite thank-you email within 24 hours of the interview.
